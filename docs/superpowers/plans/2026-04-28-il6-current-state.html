1. 한 줄 요약 + 핵심 게임플레이 루프

"끝없는 겨울. 마지막 마을의 마지막 수호자." 한국어 판타지 설원 배경의 탑다운 생존 + 베이스 디펜스. 낮에는 자원을 채집하고 동료를 영입해 마을을 키우며, 밤에는 사방에서 몰려오는 좀비 웨이브로부터 마을을 사수.

코어 루프

  1. 낮 (Day) — 자원 채집(나무/돌/고기), 동물 사냥, 떠돌이 영입, 마을 건설/수리, 모드 전환(근접/원거리/건축).
  2. 저녁 (Evening, 5s) — 화면이 검게 페이드인, "마을로 돌아가는 중..." 안내. 페이드 절정에서 플레이어 + 모든 동료/펫이 마을 중심으로 텔레포트.
  3. 밤 (Night) — 좀비 웨이브가 펜스 밖에서 무한 스폰, 활성 0 시 자동 보충. 25킬 마다 보스 강제 출현. 동료/동물/NPC 신규 스폰 정지.
  4. 새벽 (Dawn, 5s) — 골드빛 그라데이션 + "☀ 아침이 밝았다" 시네마틱. EndNight() 가 모든 좀비 청소. 플레이어/동료 풀 HP 회복, 건물 25% 수리, 자동 저장, 새 NPC 1명 스폰.

2. 사이클 시스템

  • 기본 페이즈 길이: 낮 40s, 밤 40s, 저녁 5s, 새벽 5s.
  • 레벨 스케일: Day/Night 만 1 + 0.25 × (Level-1) 곱연산. Lv 1 = 40s, Lv 5 = 80s, Lv 10 = 140s. 저녁/새벽은 고정.
  • 페이즈 길이 잠금: DayNightController._phaseLockedDuration — 페이즈 진입 시점에 ComputeDuration 결과를 캐싱. 진행 중 레벨업으로 mul 이 바뀌어도 현재 페이즈 길이는 안 변함.
  • 좀비 안전망: NightController.Update 가 매 프레임 Cycle.Phase vs CurrentPhase 비교, 어긋나면 강제 StartNight/EndNight. Zombie.Update 도 본인 phase != Night 면 자폭. 낮에 좀비가 남아있는 가능성을 다층 가드로 차단.

3. 마을 + 건물

고정 구조

게임 시작 시 VillageStarter.SpawnStarterVillage() 가 마을 중심에 모닥불 + ±5u 사각 펜스 벽 + 남쪽 가운데 슬롯에 골드 게이트 (Door) 자동 스폰. Door 콜라이더는 Player + 모든 Companion 콜라이더와 IgnoreCollision (좀비/동물은 막힘).

건물 종류 — BuildingKind enum

건축 흐름

  1. Build 모드 (탭 [3] 또는 핫바) 에서 슬롯 클릭 → _pendingBuildKind 설정.
  2. 마우스에 골드 + 커서 + 안내 배너. 좌클릭 = 확정, 우클릭/ESC = 취소.
  3. 좌클릭 시 자원 즉시 차감 후 ConstructionSite (반투명 회색 사각) 스폰.
  4. ConstructionSite.Update — 반경 2.5u 의 워커 (플레이어 + 비-Hiding/Working/Farming 동료) 수에 비례해 진행도 누적. 단독 8s, 동료 2명 ~3s.
  5. 완료 시 OnComplete 콜백이 실제 SpawnX 메서드 호출 → 진짜 건물로 교체.

비용 점증

Inflate(base, kind) = base × (1 + 0.35 × 이미 지은 같은 종류 수). 두 번째 집은 +35%, 세 번째는 +70%, 네 번째는 +105%. UI 핫바에 실시간 반영.

마을 성장도 → 건물 HP

Building.Awake 가 펜스 제외 살아있는 건물 수를 카운트해서 신규 건물 HP 에 1 + 0.12 × N (최대 ×3) 곱연산. 마을이 커질수록 새 건물이 더 튼튼해짐. 펜스는 따로 망루 수에 따라 +50%/+100%/... 적용.

4. 유닛 — 플레이어 / 동료 / 펫

플레이어

  • HP 100, MoveSpeed 5, 자동 공격 (장비 모드 따라 근접/원거리)
  • HP 회복: 4초마다 +1, 새벽 풀회복
  • 이동: WASD/방향키. InputReader Y축은 W=+1 (위) 정상.
  • 모드 탭: [1] ⚔ 근접 (Longsword) / [2] 🏹 원거리 (Bow) / [3] 🏠 건축 (Build). 건축 모드에만 BuildHotbar 표시.
  • 이동 시 발자국 눈 퍼프, 피격 시 화면 빨간 비네트 + 카메라 셰이크

동료 (Companion)

  • RecruitableNpc 가 F키 또는 영입 다이얼로그로 Companion 변환
  • HP 50, IsCombat 따라 자동 공격. 사거리 5u, 시야 9u (사거리보다 넓음).
  • Stance: 👣 Follow / 🛡 Hold / ⚔ Aggressive (HUD 토글)
  • 포메이션: 둘러싸기 X, 플레이어 facing 기준 종대로 한 줄에 2명씩 ±0.45u 흩어지며 거리 점점 멀어짐
  • follow 우선순위:
    1. 플레이어 시야 밖이면 → 적 무시하고 따라옴 (전투 중단)
    2. 플레이어 시야 안 + 적 시야 안 → 적 추격해 사거리 안에서 사격
    3. 둘 다 마을 안 → 자리 지킴 (앵커)
    4. 다른 쪽 → 가장 가까운 문 통과 2단계 경로 (좌우 정렬 후 직진)
  • 채집 중 피격 → 즉시 Mode.Follow 전환 후 반격. 채집 중 평화 시 → 적 시야 들어와도 채집 계속.
  • 자동 대피: 1초마다 TryAutoHide. (Night + IsBlizzard) 또는 (비전투 + 좀비/늑대 시야 진입) → 가까운 비-펜스/바리게이트 건물에 숨음.
  • 망루 부스트: 3.5u 내 동료의 사격 데미지 ×1.5
  • 영입 한도: FreeCapacity(12) + Houses × 4, 일일 최대 3명 (DayStarted 마다 리셋)
  • 먼 거리 콜라이더 OFF: 플레이어와 8u 이상 거리면 collider disabled (좀비/펜스 통과 가능, 정체 방지)

펫

  • SummonDog 룬 → 갈색 둥근 펫. 근접 5+lvl×3 dmg, AttackRange 1.4u, 0.55s. Rigidbody2D Dynamic + collider 0.30u (다른 유닛처럼 충돌).
  • SummonHawk 룬 → 노란 삼각 펫. 매도 근접. 5+lvl×3 dmg, AttackRange 1.2u, 0.7s. RB2D + collider 0.25u.
  • 둘 다 Player 주변 궤도, 근처 좀비 자동 추격해 슬래시 공격.

5. 적 — 좀비 + 보스

좀비 (Zombie)

  • NightController 가 밤에만 스폰. PickOutsideVillageSpawnPos 가 펜스 사각(±5u + 1u 여유) 안이면 외부로 밀어냄 — 마을 안에서 절대 안 뜸.
  • 일자별 강화 (per-night scaling): HP × 1 + 0.12 × (day-1), damage +(day-1), speed × 1 + 0.02 × (day-1) (캡 1.5×).
  • 변종:
    • Day 3+ Fast (오렌지, 0.8x scale, 1.6× speed, 60% HP)
    • Day 7+ Tank (보라, 1.35x scale, 0.6× speed, 220% HP, +4 dmg)
  • 경로 차단 시 건물 부수기: Linecast 로 플레이어/동료까지의 경로에 Building 콜라이더 있으면 그 건물을 우선 공격해 길 확보.
  • 상태이상: Poison(누적 DoT), Slow(이속 ½), Lightning chain, Detonate(처치 시 폭발)

보스

  • Day 5/10/15 (5의 배수) 시 BossNight: 3초 카운트다운 + ⚠ 보스 출현 배너 + 카메라 셰이크.
  • 킬 누적 보스: 임의 밤에서도 KillsThisNight ≥ 25 도달 시 그날 보스 미출현이면 강제 소환.
  • HP 60 + day × 8, 공격 +6+day-1, 변종 색 보라.

웨이브 시스템

  • BaseWaveCount 6 + (day-1) × 3, 눈보라 시 ×2, 매 활성 0+대기 0 도달 시 Base + KillsThisNight/5 로 무한 보충.
  • 스폰 간격 0.6s (눈보라 0.36s), MaxActive 40.
  • 눈보라(20% 확률): 플레이어 모닥불 반경 밖이면 DPS 2 동상 데미지, 동료도 대피 트리거.

7. 전투 + 룬 시스템

무기

WeaponCatalog — 2종만: Longsword (근접, ProjectileSpeed=0, 16 dmg, 1.8u, 0.7s) / Bow (원거리, 10 dmg, 7.0u, 1.1s, 투사체 10 speed). 모드 탭으로 자동 전환.

투사체

Projectile.Awake 가 자체 sprite 생성 (ColorFallback 의존 X — 실행 순서 문제 회피). 큰 흰 원 (PixelSize 48-64) + 1.7~2.4× soft glow halo + 0.025s 간격 ghost trail. sortingOrder 50, scale 0.9. Pierce 룬 시 ×N 관통.

룬 시스템

레벨업 시 3장 카드 모달 (920×380 큰 사이즈) 에서 1장 선택. 각 룬 최대 3중첩 (1=기본, 2=강화, 3=마스터). 같은 원소(독·얼음·번개) 중 2개 마스터하면 시너지 ×1.5.

단순 능력치 룬은 풀에서 제거됨 (DamageUp/FireRateUp/HpUp/RangeUp/MoveSpeedUp enum 만 보존, 풀에는 없음). 모든 선택지는 효과 기반:

8. 자원/채집/수리/영입

자원

Wood / Stone / Meat / Food / Frostbloom 5종. 각 캡 50 (Storage 1개당 +50). ResourceStore 가 Add/Spend/IncreaseCap 관리.

채집 (Gather)

  • 나무: 9s, +4 Wood. 돌: 12s, +3 Stone. 벌목/채굴 두 종류 다 월드 버튼 (220×52 큰 사이즈).
  • 버튼 표시 조건: 노드 3.5u 안에 플레이어 또는 동료 1명 이상.
    • 플레이어 근처 → 본인 채집 시작 + 근처 동료 (최대 2명) 합류, 라벨 (나 + 동료 N)
    • 플레이어 멀고 동료만 → 가장 가까운 동료 최대 2명, 라벨 (동료 N)
  • 플레이어 E키 채집 → 자동 합류: GatherController 가 0.5s 마다 4u 안 Follow 동료를 최대 2명 자동 AssignGather.
  • 완료 시 자원 + 녹색 "+N 자원" 토스트 + DeathPoof + 픽업 사운드.

수리 (Repair)

손상된 건물 (HP < MaxHp) 3.5u 안에 플레이어 또는 동료 있으면 🔨 수리 버튼 표시. 클릭당 1 Wood 소모, MaxHp의 20% 회복. 도배 클릭으로 풀 수리. 새벽엔 자동 25% 회복도 별도.

영입 (Recruit)

  • 출현 조건: 낮 페이즈에만 스폰. 전 세계 동시 RecruitableNpc ≤ 3 (캡), DailyNpcIntervalSec 30s 마다 마을 6.5~9u 외곽에 1명 + DayStarted 시 1명 추가.
  • 아키타입 5종: 사냥꾼 (활), 전사 (검 9 dmg), 치유사 (원거리), 농부 (비전투), 방랑객 (밸런스).
  • 한도: VillageCapacity = FreeCapacity(12) + Houses × 4 + 일일 최대 3명 (DayStarted 마다 리셋).
  • F키 또는 영입 대화창. 한도 초과 시 버튼 disabled "건물 더 필요".

9. UI 레이아웃

모든 HUD 는 IMGUI 기반 (UiTheme 골드/네이비 테마). Don't Starve / Vampire Survivors / 7DTD 참고한 모서리 카드 + 하단 핫바 구조:

상단

  • TOP-LEFT WaveStanceBar (320×120) — 페이즈별 컨텍스트 라인 (좀비/대기, 식량 부족, 평온한 낮) + 동료 스탠스 토글 [하단]

11. 진행/레벨/업적

  • XP: 좀비 처치당 +1, XpToNext = 8 + Level × 4. Lv 1→2 = 12, 2→3 = 16, 3→4 = 20. 한 밤 평균 28킬 → ~2렙업.
  • Achievements (9개): 첫 사냥, 25/100킬, 첫/5명 동료, 3/7/15일 생존, 점수 300. 토스트로 표시.
  • SaveLoad: JSON 파일 (currentDay + ResourceSnapshot), 새벽마다 자동 저장 + ESC 즉시 저장.
  • Score: (Day-1) × 10 + Kills × 3 + MaxComp × 5 - Lost × 4. 사망 화면에서만 표시.
