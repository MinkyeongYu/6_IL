요약

현재 상태는 "실행 가능하지만 대부분 placeholder"입니다. 실제 게임용 이미지/오디오 파일은 아직 없고, 문서용 프리뷰 이미지만 존재합니다. 따라서 다음 작업의 우선순위는 P0 시각 리소스 교체, 피드백 이펙트, 오디오 MVP, 앱 아이콘 순서가 적절합니다.

권장 관리 방식: 각 행을 이슈/스프레드시트 한 줄로 옮기고, 담당자, 진행 상태, 원본 파일 링크, 게임 적용 여부 칼럼만 추가하면 됩니다. 파일명은 아래 제안명을 기준으로 통일하고, 원본은 art/source/, 게임 내보내기는 public/assets/ 또는 src/assets/로 분리하는 것을 권장합니다.

아트 납품 체크리스트

  1. 원본 파일과 게임 내보내기 파일을 모두 제출합니다. 예: art/source/characters/player_survivor.aseprite, public/assets/characters/player_survivor.png.
  2. 애니메이션 스프라이트는 프레임 크기, 방향 순서, 피벗 기준을 파일명 또는 동봉 JSON에 명시합니다.
  3. 건물은 정상/손상/파괴 상태가 같은 풋프린트에서 흔들리지 않게 정렬합니다.
  4. UI 아이콘은 32px에서 식별 가능해야 하며, 64px 확장 버전도 같이 관리합니다.
  5. 밤/눈보라 리소스는 어두운 배경 위에서 실제 플레이 스크린샷에 얹어 판독성을 확인합니다.
  6. 완료 판정은 “파일 존재”가 아니라 게임 화면에 적용되어 비율, 충돌, 가독성이 맞는 상태를 기준으로 합니다.
