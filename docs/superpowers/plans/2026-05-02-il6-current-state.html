1. 한 줄 요약 + 코어 루프

"끝없는 겨울. 마지막 마을의 마지막 수호자." 한국어 판타지 설원 탑다운 생존 + 베이스 디펜스. 낮에는 자원 채집 + 동료 영입 + 마을 확장(펜스도 자동으로 자라남), 밤에는 좀비 웨이브 사수.

코어 루프

  1. 낮 (Day) — 자원 채집(나무/돌/고기), 동물 사냥, 떠돌이 영입, 건물 짓기 + 자원으로 모닥불 연료 보충, 동료 레벨업.
  2. 저녁 (Evening, 5s) — 화면 검게 페이드 → 마을로 자동 텔레포트 (플레이어 + 동료 + 펫 모두).
  3. 밤 (Night) — 좀비 웨이브 무한 보충, 25킬마다 보스 강제, Day 4+ 부터 원거리 Archer 좀비. 마을 한기는 모닥불 연료가 살아있을 때만 막아짐.
  4. 새벽 (Dawn, 5s) — 골드빛 시네마틱 + "☀ 아침이 밝았다", EndNight 모든 좀비 청소, 풀 HP 회복, 건물 +25% HP, 새 NPC 1명 스폰, 자동 저장.

2. 사이클 시스템

  • 기본 페이즈 길이: 낮 40s, 밤 40s, 저녁 5s, 새벽 5s.
  • 레벨 스케일: Day/Night 만 1 + 0.25 × (Level-1) 곱연산. Lv 1 = 40s, Lv 5 = 80s, Lv 10 = 140s. 저녁/새벽 고정.
  • 페이즈 길이 잠금: DayNightController._phaseLockedDuration 캐싱 — 진행 중 레벨업해도 현재 페이즈 길이는 안 변함.
  • 밤 진입 안전망: NightController.Update 가 Cycle.Phase ↔ CurrentPhase 동기화 (이벤트 누락 케이스 보장). Zombie.Update 도 본인 phase != Night 면 자폭.

3. 마을 + 건물 (10종)

고정 구조

게임 시작 시 VillageStarter.SpawnStarterVillage 가 마을 중심에 모닥불 + ±5u 사각 펜스 + 남쪽 게이트 (Door, Player+Companion 통과). 건물을 지을 때마다 펜스 사각이 자동으로 확장 — 비-펜스 건물 1채당 +0.4u, 최대 14u 까지. 새 외곽에 fence ring + 게이트 생성, 0.4u 안에 기존 펜스 있으면 중복 스킵.

건물 종류

14. 코드 구조 — 리팩터 후 ⭐

이번 갱신은 4개의 핵심 리팩터로 코드 베이스가 한 단계 단단해졌다.

14.1 AnimalAi 추상 부모

// Assets/Scripts/Animals/AnimalAi.cs
public abstract class AnimalAi : MonoBehaviour {
    public int MaxHp = 1;
    public int CurrentHp { get; protected set; }
    public bool IsDead => CurrentHp <= 0;
    public float RegenIntervalSec = 6f;
    protected Rigidbody2D _rb;
    protected SpriteRenderer _sr;
    protected Transform _player;

    public void InitHp(int hp) { MaxHp = hp; CurrentHp = hp; }
    public virtual void TakeDamage(int amount) { /* HP감소, 플래시, 사망 시 가죽 */ }
    protected abstract void DoBehavior();    // 자식이 채워야
}
  • DeerAi = 도주형 — DoBehavior() 에서 플레이어 시야 진입 시 opposite 벡터로 도주.
  • WolfAi = 포식자형 — 늑대(무리)/곰(단독)/맘모스(정적) 모두 사용. DoBehavior() 에서 시야 안 가장 가까운 hostile 추격 + 근접 슬래시.

15.2 Generate All Prefabs 메뉴

Unity 에디터에서 IL6 → Tools → Generate All Prefabs 한 번 클릭.

  1. 각 카테고리 폴더 자동 생성 (없으면).
  2. 건물 10종 — Campfire, Fence, Gate, Barricade, House, Storage, Farm, Watchtower, Infirmary, HuntersHut.
  3. 동물 7종 — ProceduralSpawner._animals 순회하며 CreateOneAnimal 으로 1마리 생성 → prefab 저장.
  4. 동료(NPC) 5종 — _npcArchetypes 순회하며 CreateNpcFromArchetype 으로 1명 생성 → prefab 저장.
  5. 씬에선 즉시 DestroyImmediate, 자산만 디스크에 남는다.
  6. 합계 22 prefabs 생성 완료 토스트.

현재 prefab 들은 ColorFallback 기반 절차 스프라이트를 들고 있어, 외주 아트 받기 전이라도 바로 검수 가능. 외주 sprite 가 도착하면 prefab 의 SpriteRenderer.sprite 슬롯만 교체 + ColorFallback disable 시 끝.

15.3 외주 아트 받기 워크플로우 (제안)

  1. 외주에 Assets/Prefabs/ 의 prefab들 스크린샷 + Tint 색 + 크기 스펙(localScale.x/y) 전달.
  2. 받은 .png 들을 Assets/Art/Sprites/{Animals|Buildings|Companions}/ 에 복사. Pixels Per Unit = 32 권장 (현재 PixelSize 와 일관).
  3. 각 prefab 열어 SpriteRenderer.sprite 슬롯에 교체.
  4. ColorFallback 컴포넌트는 비활성 (gameObject.Component 의 체크박스 OFF) — 외부 sprite 가 우선.
  5. 씬 재생해 색감 확인 + 필요 시 Tint 만 조정.
