A. 아키텍처 개요 + 진입점

BootScene (Unity scene)
  └─ BootController (MonoBehaviour, DefaultExecutionOrder=-100)
       ├─ creates GameObject "GameSession" + adds GameSession component
       │   └─ GameSession.Awake → ResourceStore + DayNightController + AchievementManager
       └─ SceneManager.LoadScene("SnowfieldScene")

SnowfieldScene
  └─ SnowfieldController (MonoBehaviour)
       ├─ Player(PlayerController) reference 인스펙터 연결
       ├─ ChunkManager / VisionMask / Camera 인스펙터 연결
       ├─ Start: VillageStarter.SpawnStarterVillage(center)
       ├─ Start: 마을 14u 안의 잔여 동료/NPC/동물 정리
       ├─ Start: DayStartedPayload 구독 → ProceduralSpawner.SpawnDailyVillageNpc
       └─ Update: Cycle.Update(dt) + 마을 한기 데미지 4s 마다

GameSession (DontDestroyOnLoad 싱글톤)
  ├─ Resources : ResourceStore
  ├─ Cycle     : DayNightController
  └─ AchievementManager 동일 GameObject 에 자동 부착

모든 게임 상태는 GameSession.Instance 싱글톤 1개를 통한다. 페이즈 전환은 EventBus 의 DayStartedPayload / EveningStartedPayload / NightStartedPayload / DawnStartedPayload 4종 이벤트로 모든 시스템이 동기화된다.

B. 코어 — Session / Cycle / Config

Assets/Scripts/BootController.cs
  • Boot 씬에 단 한 번 부착. DefaultExecutionOrder(-100) 으로 다른 어떤 Awake 보다 먼저 실행.
  • 역할: GameSession GameObject 생성 + SnowfieldScene 로드.
  • 의존: 없음 (가장 진입 시점).
Assets/Scripts/GameSession.cs
  • 씬 간 영속 싱글톤. GameSession.Instance로 어디서나 접근.
  • 주요 멤버: Resources (ResourceStore), Cycle (DayNightController), TotalKills, CompanionsLost, MaxCompanionsAtOnce, Score.
  • OnDayStarted(int day): 식량 소비, 동료 사기 데미지, 새벽 풀 회복, 자동 저장.
  • OnZombieKilled(): 킬 카운트 + NightController 알림 + 가장 가까운 동료에 +1 XP 분배.
  • HardReset(): 세이브 삭제 + 씬 리로드.
Assets/Scripts/Cycle/DayNightController.cs
  • 4 페이즈(Day/Evening/Night/Dawn) 머신. 외부 dt 입력 받아 진행.
  • _phaseLockedDuration 캐싱 — 페이즈 진입 시점 길이가 고정, 진행 중 LevelDurationMul 변경되어도 현재 페이즈 길이 안 변함.
  • 페이즈 끝나면 EventBus 로 페이로드 emit.
  • LevelDurationMul = 1 + 0.25 * (Level-1) — Day/Night 만 적용.
DurationOf 가 NaN 또는 0 이 되면 cycle.Update 안 while 루프가 무한 돌 수 있음. Mathf.Max(1f, LevelDurationMul) 로 클램프 되어 있으니 안전.
Assets/Scripts/Config/BalanceConfig.cs
  • ScriptableObject — 모든 밸런스 수치 단일 소스.
  • 주의: 코드의 default 값과 Resources/BalanceConfig.asset 값이 다를 수 있음. 런타임에는 asset 이 우선. 새 필드 추가 시 두 곳 다 업데이트.
Assets/Scripts/Config/GameConstants.cs
  • VillageCenterX/Y, VillageGridSize, TileSize 등 컴파일 타임 상수.

  • HpUp 룬 적용 시 MaxHp 갱신 (구 룬 호환).

Assets/Scripts/Combat/PlayerProgression.cs

  • PickThreeOffer(seed) — 가중치 풀에서 마스터 안 된 룬 3장 결정적 추첨.
  • ApplyRune(kind): 스택 +1 + Recompute() (DamageMul/HpBonus 갱신) + 즉시 효과 (펫 소환/자원 보급).
  • 같은 원소 2개 마스터 시 SynergyMul = 1.5.
  • 레벨업마다 GameSession.Cycle.LevelDurationMul = 1 + 0.25*(Level-1) 세팅.
Assets/Scripts/Combat/DamageCalc.cs
  • 정적 함수 Compute(in Input) → int. 결정적 (RNG 외부 주입). 크리티컬 + 방어력 적용.

E. 동료

Assets/Scripts/Companions/Companion.cs
  • Mode enum: Follow / Working / Farming / Hiding / Fleeing.
  • Stance enum: Follow / Hold / Aggressive (HUD 토글).
  • HP 50, IsCombat 따라 자동 공격. Sight 9u, Attack 5u.
  • follow 우선순위 (ComputeFollowVelocity): ① Hold 스탠스 → 앵커 / ② 플레이어 시야 밖 → 따라옴 (적 무시) / ③ 시야 안 + 적 → 추격 / ④ 둘 다 마을 안 → 자리 지킴 / ⑤ 다른 쪽 → 문 통과 2단계.
  • 채집 중엔 전투 X, 단 TakeDamage 시 즉시 Working/Farming 탈출 후 반격.
  • TryAutoHide 1초 폴링 — 눈보라 또는 비전투+적 시야 → 가까운 비-펜스/바리게이트 건물에 숨음.
  • 개별 XP/장비: Xp / Level / XpToNext = 4 + L*3, EffectiveDamage = Damage × (1+0.10×(L-1)) × 장비 배수. 장비 enum: SharpenedBlade/Crossbow/ThickFur/ToolKit. 부착 UI 미구현.
  • 망루 부스트 ×1.5, AllyBoost 룬 부스트 둘 다 데미지에 곱연산.
  • 플레이어와 8u 이상 떨어지면 collider 비활성 (정체 방지).
Assets/Scripts/Companions/RecruitableNpc.cs
  • F키 또는 영입 다이얼로그 → Recruit() 가 같은 GameObject 에 Companion 컴포넌트 추가.
  • 마을 수용 한도: FreeCapacity(12) + Houses × 4. 일일 한도 X.
  • 아키타입 5종 (사냥꾼/전사/농부/아이/노인) 은 ProceduralSpawner._npcArchetypes 에 정의.

  2. PlayerProgression:
